icode4.fun-api

What is this: A system for processing large email archives and building collections for eDiscovery research. Used to build the Avocado email research collection (published by the Linguistic Data Consortium).

sample code: pst extraction script

Developed: 2011-2012 (E-Discovery Lab, University of Maryland, College Park)

Technology: System written mostly in Python with some Java. MySQL as the database. Relies on the libpst library for extracting PST a Included a web-based admin interface for exploring the dataset, written in PHP and JavaScript.

My part

Initial datset: an email archive of a defunct company

I was hired to build the collection. My work fell into two categories: (1) developing the processing pipeline (and designing the xml-based format for the collection?) (with help from William, Mosaab, and Doug, my PI)

The initial data set consisted of 283 Microsoft PST files (66GB)

My task: extract, arrange, catalogue, redact into organized corpus to be shared with research community

- Processing Stages:

  1. PST file extraction
  2. File deduplication
  3. MIME-type identification
  4. Archive extraction
  5. Text extraction
  6. distribution?

Lessons learned

Repeatability of data processing steps - hense, a pipeline. With any data set, data processing may require many, many steps. Automating the sequence of these steps simplifies everything: you can tweak and rerun as often as you like. With a dataset of this size, automating the pipleine is a must...
